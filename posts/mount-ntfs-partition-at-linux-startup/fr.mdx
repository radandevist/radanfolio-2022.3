J'ai à la fois Linux et Windows (dual boot) installés sur mon PC, donc le disque est partitionné en trois:
- le système d'exploitation Linux,
- Windows 10, et
- une partition pour partager des fichiers entre les deux systèmes d'exploitation (car nous ne pouvons pas parcourir le système de fichiers Linux à partir de Windows).

Chaque fois que je démarre sous Linux et que je veux explorer mes autres partitions, je dois d'abord les monter. Mais en fait, nous pouvons les faire monter automatiquement au démarrage.

Ouvrez le fichier `/etc/fstab` avec votre éditeur préféré:

```bash:terminal
sudo nano /etc/fstab
```

vous verrez quelque chose comme ça (cela peut légèrement varier selon la distribution):

```bash:/etc/fstab
# /etc/fstab: static file system information.
#
# Use 'blkid' to print the universally unique identifier for a device; this may
# be used with UUID= as a more robust way to name devices that works even if
# disks are added and removed. See fstab(5).
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>                 
UUID=b57600bf-231d-4da2-bd44-e392eag85627 /              ext4    defaults,noatime 0 1
```

Comme vous pouvez le voir, vous aurez besoin de l'UUID de la partition que vous souhaitez monter automatiquement. Pour l'obtenir, exécutez simplement dans un terminal:

```bash:terminal
sudo blkid
```

Une liste des appareils de votre machine s'affichera avec certaines de leurs informations, y compris leurs UUID. Sélectionnez l'UUID souhaité et revenez au fichier `/etc/fstab`.

En fin de fichier modifier la ligne:

```bash:/etc/fstab
UUID=YOUR_UUID /path/to/mount/point  auto,nofail,uid=1000,gid=1000,utf8,umask=022,defaults 0 0
```

N'oubliez pas de remplacer uuid, le chemin du point de montage et le système de fichiers de la partition.

### Informations sur le système de fichiers de partition ntfs:

Personnellement, je voulais monter une partition explorable depuis Windows, c'est-à-dire une partition <span className="text-brand1">ntfs</span>. Sous Linux, la clé correspondante pour monter ce type de système de fichiers est `ntfs-3g`. Vous aurez besoin du package éponyme installé sur votre système d'exploitation Linux. Il est déjà préinstallé sur presque toutes les distributions Linux, mais vérifiez-le cependant au cas où vous l'auriez supprimé par inadvertance. J'ai eu un peu de mal à le découvrir la première fois que j'ai utilisé Arch, donc je pense que cela peut aider quelqu'un à le mentionner.

Je ne couvrirai pas les options de montage dans cet article, mais vous pouvez en savoir plus à leur sujet [ici](https://wiki.archlinux.org/title/fstab).
